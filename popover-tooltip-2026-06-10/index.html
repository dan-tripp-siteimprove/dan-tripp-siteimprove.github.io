C the third letter of the alphabet

This code was originally from MDN (main MDN article is here, and specifically I used this demo here). Then I modified it. It implements a tooltip via the HTML popover attribute, specifically a "hint popover" i.e. popover="hint".

Features/limitations of this tooltip:

  1. The tooltip is meant to be added to an interactive widget eg. button or link. Maybe it works for other widgets too - I don't know.
  2. This tooltip is meant to contain a small amount of text. Not a lot of text. Not any interactive widgets or images. Maybe those work, or could be easily made to work - I don't know.
  3. The tooltip appears on mouse hover or keyboard focus. It disappears on mouse out or blur. Consequently there will be max one tooltip visible at a time.
  4. The tooltip's text is included in the accessible name of the widget. So keep the tooltip text short. We don't use accessible description AKA aria-describedby.
  5. The tooltip text is always in the a11y tree (as part of the accName of the target widget), regardless of whether the tooltip is currently visible or not. So if you, say, use a screen reader and arrow nav to a button, you'll hear the tooltip text.
  6. The tooltip can be dismissed with the ESCAPE key.
  7. If the target widget is near the edge of the viewport: the tooltip will appear within the viewport.
  8. The tooltip will almost not overlap its target widget. The problem is on Firefox, and it happens because of something unsupported about anchor positioning, I think. To do: fix that.
  9. Not much Javascript is needed.
  10. The behaviour "tooltip's text is included in the accessible name of the widget" is good as far as it goes. But I wonder if this overall approach - i.e. tooltip text in the accName, visible tooltip text aria-hidden - is not ideal. Maybe we should not aria-hide the visible tooltip text. I think we'd need to advertise the existence of the tooltip to screen readers somehow [2], and expect them to TAB to the widget (to make the tooltip appear / get added to the a11y tree) then use arrow key nav down to hear that tooltip text. [1] NVDA/Firefox behaviour here is different from Chrome. On Firefox, the visible text and the tooltip text are in different "sentences". I don't know whether Firefox is better or worse than Chrome in this regard. [2] Probably not via accDesc b/c I don't trust it.
  11. Possible bug: since the tooltip-contents-as-popover are aria-hidden, I wonder if there is a bad user experience possible here. Something like a sighted user using NVDA with "mouse tracking" on.
  12.
