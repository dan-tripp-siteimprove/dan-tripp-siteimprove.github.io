MT Tools

Introduction

"MT" stands for "Manual Testing". These are tools for manual web accessibility testing. Currently these tools are all in "bookmarklet" form, as opposed to browser extensions or something else. All of them except the "target size" bookmarklet were forked from Paul Adam's bookmarklets by Siteimprove in November 2023. (Here is Paul's install page and Paul's github repo.) Many thanks to Paul for his great bookmarklets. Our main reason for forking them was to make them work on certain sites which used a restrictive Content Security Policy or shadow DOM. Our main reason for creating our own "target size" bookmarklet was because, of all the public implementations of similar bookmarklets that we knew of at the time (November 2023), none of them had a mouse cursor that handled different browser zoom levels.

These bookmarklets work on some iframes. I think that they work on "same-origin" iframes and don't work on "cross-origin" (eg. third party) iframes.

Handling of shadow DOM handling varies by bookmarklet.

Here is the source code on github for these tools. To see this source code, you might need to have permissions as a member of the Siteimprove github "organization". If you don't, you might get a 404 error if you follow that github link.

Contact: Dan Tripp (dtr at siteimprove.com).


Installation and usage

Screen-reader users

These instructions could also be useful for sighted keyboard-only users, if you ignore certain parts. These instructions were written based on a first-hand experience with NVDA on 2024-09-25. I witnessed (but didn't do myself) a JAWS installation around the same time. With JAWS, the process is similar, I think - in particular, the "Tab (don't arrow)" part. If you arrow (rather than tab) to the link, with either NVDA or JAWS, you will have a bad time.

Chrome installation

  • Show the bookmarks toolbar
    • Technically ctrl+shift+b will show/hide the bookmarks toolbar, but it won't give you any screen reader announcement as to whether it's shown or hidden. so we'll do it another way instead.
    • Press F10 (to move focus to the chrome "customize" button). Arrow down to "bookmarks and lists", then within that submenu, arrow down to "show bookmarks bar". If instead you only find "hide bookmarks bar", that means that the bookmarks bar is already shown.
  • Installing this bookmarklet requires an unusual process: we're going to create a dummy bookmark then edit it.
  • Bookmark the page you're reading now by pressing ctrl+d.
  • Set the folder to be "bookmarks bar"
  • Tab (don't arrow) to the link on this page for the bookmarklet in question eg. "ARIA (2023-11-24 version)"
  • Shift+F10, then use down arrows until you get to "copy link address"
  • Press F6 a few times - until you get to the "bookmarks toolbar" region.
  • Tab (or use right/left arrow keys) to get to the dummy bookmark that you just made. It will probably have the same name as the title of this web page that you are reading: "MT Tools" or something like that.
  • Shift+F10 to open the context menu for that bookmark. Arrow down to "Edit".
  • In the "name" field, enter the name of the bookmarklet (eg. "ARIA").
  • In the "URL" field, paste the javascript code, which should still be in your clipboard, since the "copy link address" step.
  • BEWARE of deviating from the steps above in this way: re-using an old bookmark instead of making a new dummy bookmark. I say this because I have seen some buggy behaviour regarding pasting long code like this into the bookmark's "value" field, where it silently didn't update. If you use a new dummy bookmark, any problem during pasting will be obvious. If you re-use an old bookmark, it won't.

Chrome usage

  • Press F6 a few times, until you enter the region "bookmarks toolbar".
  • Tab or use right/left arrow keys until you get to your bookmarklet (eg. "ARIA").
  • Press ENTER to run the bookmarklet.

Firefox installation

  • Show the bookmarks toolbar
    • Technically ctrl+shift+b will show/hide the bookmarks toolbar, but it won't give you any screen reader announcement as to whether it's shown or hidden. So we'll do it another way instead.
    • alt+v (for the "View" menu) > toolbars > bookmarks toolbar > always show.
  • Tab (don't arrow) to the link on this page for the bookmarklet in question eg. "ARIA (2023-11-24 version)"
  • Shift+F10, then down arrow until you get to "Bookmark Link..."
  • Save

Firefox usage

  • Press F6 to get to a region called "navigation toolbar".
  • Tab a few times, until you get to the "bookmarks toolbar".
  • Then use right arrow keys to get to your bookmarklet (eg. "ARIA").
  • Press ENTER to run the bookmarklet.

Mouse users

Installation

This works in any browser, probably.

  • Find the link on this page for the bookmarklet in question eg. "ARIA (2023-11-24 version)"
  • Drag that link to your bookmarks toolbar.

BEWARE if you, as a mouse user, use any installation process other than the above drag-and-drop process. Specifically: if you re-use an old bookmark and paste the code into it. I say this because I have seen some buggy behaviour regarding pasting long code like this into a bookmark's "value" field, where it silently didn't update. If you use the drag-and-drop method above, you will never have that problem. If you re-use an old bookmark, you might.

Usage

  • Click the link in your bookmarks toolbar.

Target Size bookmarklet

Info

  • This bookmarklet will help you test for target size, for both the AA and AAA success criteria. i.e. both SC 2.5.8 (AA, 24 px) and SC 2.5.5 (AAA, 44 px).
  • This bookmarklet doesn't make a judgement on whether an element is bigger than 24x24 or not. It used to, around version v0.1, via blue squares and red circles. Now it doesn't, because those judgements were wrong as often as they were right. Instead, this bookmarklet tries to help you make that judgement.
  • This one is a Siteimprove original. It wasn't forked from the Paul Adam bookmarklets.
  • Known bug: on some pages there will be a "dead zone" for the mouse cursor. i.e. this bookmarklet's mouse cursor won't work on top of certain elements. One example that will trigger this "dead zone" bug is a native HTML date picker popup.
  • Known bug: it's slow on big pages. It might freeze for 30 seconds when you run it or refresh this bookmarklet. Work-around: disable "Element decorations" in the bookmarklet's options menu.
  • Known bug: Regarding the 'bounding boxes' (which are drawn as part of 'element decorations') for links that are both 1) in paragraphs, and 2) are line-wrapped: these bounding boxes will mislead you. They'll show up as much bigger than the link really is. This is a moot point for the success criterion, because of the its exception for inline targets. But it's still visual clutter that is sub-optimal for you, the user of this bookmarklet.
